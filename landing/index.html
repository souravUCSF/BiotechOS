One engine, three things it does

Triage the inbox. Verify the numbers. Design the next molecule.

Constantly monitoring your inbox for data, quotes, legal docs, POs and more. BiotechOS hands you an action you can approve — and never commits anything silently.

Triage

Inbox → an action you approve

One classifier tags every email and attachment — Quote · Invoice · Legal · Data · Other — then routes it to a specialized processor that returns a reviewable action. Approving an item is the only thing that mutates state.

engine/classifier.py
Verify

It re-derives the vendor's numbers

Analyzers recompute rather than transcribe: a 4-parameter logistic refit re-derives IC50, checks recovery and efflux bands, and flags any discrepancy against the vendor's reported value before it ever lands in the database.

engine/curvefit.py
Design

Closes the design→make→test loop

Boltz-2 co-folds the target with a ligand, maps protein–ligand contacts with ProLIF, and generates de novo analogs to feed the next round — about $0.05 a complex.

engine/structure.py · boltz.py

Run it

Local-first. One SQLite file per program.

The whole OS runs on your machine — and every processor runs end-to-end even without an API key.

# clone and run (see README for exact commands)
git clone <your-repo-url> biotechos && cd biotechos

# backend — FastAPI on :8010
cd backend && uv sync && uv run uvicorn biotechos.api.main:app --port 8010

# frontend — Next.js on :3000
cd frontend && npm install && npm run dev
